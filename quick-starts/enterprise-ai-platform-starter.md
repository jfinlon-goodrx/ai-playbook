## First Three Wins

1. Create a read-only context workflow across Gleen, Jira, Confluence, and GitHub.
2. Standardize one story-shaping and one story-to-code workflow.
3. Close the loop by writing useful completion context back to Jira and Confluence.

## Working Rules

- keep one source of truth per type of information
- start with read-only MCP or API access
- make write actions structured and auditable
- store durable knowledge in Confluence or the repo, not only in chat
- keep repo context strong so external systems support the codebase instead of replacing it
