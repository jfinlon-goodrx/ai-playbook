# Frontend and UI Starter

## Goal

Help a team use AI to build better UI faster without creating inconsistent, inaccessible, or hard-to-review frontend changes.

## First Read

- `../guides-by-platform/UI/Figma-And-AI-Frontend-Workflow.md`
- `../guides-by-platform/UI/Frontend-Tooling-Recommendations.md`
- `../prompts/by-task/frontend/README.md`
- `../prompts/by-task/frontend/ui-from-figma.md`

## First Three Wins

1. Turn one Figma design into a component plan before implementation starts.
2. Standardize one frontend stack and one component-system approach.
3. Add screenshot, responsive, and accessibility evidence to UI PRs.

## Working Rules

- do not generate UI from screenshots alone
- reuse existing components before creating new ones
- require loading, empty, error, and validation states
- review accessibility and responsive behavior every time
- keep design-system choices explicit in prompts
