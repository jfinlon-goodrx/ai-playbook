# Developer Starter

## Goal

Get a developer productively using AI in a real repo without skipping review, tests, or architecture discipline.

## First Read

- `../getting-started/03-project-context.md`
- `../getting-started/04-when-to-use-ai.md`
- `../prompts/by-task/backend/README.md`
- `../prompts/by-task/qa/test-generation.md`

## First Three Wins

1. Use AI to plan a small vertical slice from a real story.
2. Use AI to generate tests or extend test coverage.
3. Use AI to draft a PR summary and review checklist.

## Working Rules

- point the assistant at real files first
- ask for assumptions and risks before implementation
- keep generated changes reviewable
- prefer one coherent slice over scattered code snippets
- run tests and review output like any other engineering artifact
