# Developer Workflow

This section groups the practical assets that help a team work well with AI day to day.

It is intentionally focused on repeatable engineering workflow, not generic productivity advice.

## What This Section Is For

- teaching a team how to structure AI-assisted implementation work,
- reducing context loss across sessions,
- keeping reusable rules, prompts, and skills close to the work,
- enabling parallel work safely with worktrees and clear task boundaries.

## Suggested Adoption Order

1. Standardize one AI coding tool and one project-context pattern.
2. Save reusable prompts, rules, and skills in the repo instead of keeping them in chat history.
3. Use worktrees for parallel tasks, experiments, and safer agent-assisted changes.
4. Add shared planning, review, and testing workflows once the basics are stable.
