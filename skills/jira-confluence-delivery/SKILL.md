# Jira and Confluence Delivery

## Quick Start

When this skill applies:

1. Read the Jira ticket, comments, linked work, and relevant repo context.
2. Translate vague work into testable delivery language.
3. Separate product ambiguity from technical ambiguity.
4. Produce artifacts that developers, QA, product, and managers can all use.

## Workflow

### 1. Understand the work item

Extract:
- business goal,
- acceptance criteria,
- dependencies,
- blockers,
- release or coordination needs.

### 2. Decide the artifact

Use the right output shape:
- story refinement,
- sub-task breakdown,
- implementation handoff,
- Confluence ADR,
- rollout or release note.

### 3. Write for downstream use

Make the artifact useful for:
- implementation,
- review,
- QA,
- stakeholder visibility,
- future onboarding.

Do not leave key assumptions implicit.

### 4. Close the loop

Call out:
- missing decisions,
- evidence of completion,
- links that should be updated,
- and what knowledge should be preserved in Confluence.

## Good Output

Good output is:
- concise,
- testable,
- role-aware,
- and easy to paste into Jira or Confluence without heavy cleanup.
